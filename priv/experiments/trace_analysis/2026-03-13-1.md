## Concrete Improvement Suggestions

### Priority 1 — Act on the unacknowledged feedback: explore strategy combinations

The feedback "Consider that you can combine strategies" has sat unacknowledged for multiple sessions. This is the highest-leverage untested direction given the current state:

- **MaGatedRotation + BbRsiReversion**: MaGatedRotation is all-or-nothing (fully in or flat per asset). BbRsiReversion could handle the flat periods by catching oversold bounces. Combined, the system would be mostly-in-market with two uncorrelated signal sources.
- **MaGatedRotation as allocation layer on RegimeDetector**: Use MaGatedRotation's cross-sectional ranking to weight which pairs RegimeDetector trades more aggressively.

The next hypothesis should explicitly test one of these combination architectures. Mark the feedback as acknowledged first.

### Priority 2 — Address the drawdown problem before claiming the momentum cluster is solved

Both passing experiments have severe max drawdowns (47-73% training, 55-58% validation). The pass criteria currently requires `Sharpe > baseline OR max_drawdown < 40%` — if Sharpe data is null (not computed), the drawdown gate may not be enforced. The MaGatedRotation variants are passing with 58%+ validation drawdowns, which would be unacceptable in production.

**Concrete action**: add a stop-loss or drawdown circuit-breaker to MaGatedRotation, then re-run. A trailing stop at -15% per asset before rotation would reduce drawdown without killing the momentum signal's core edge. If it still passes, the strategy is truly robust.

### Priority 3 — Diversify into mean_reversion or carry mechanisms

The last 6/7 experiments are momentum variants. Mean reversion (not yet in `mechanisms_seen`) and carry are completely unexplored. Given:
- BbRsiReversion already exists in the codebase and has known validation behavior
- PostShockReversal showed a real per-trade edge (WR 52%, PF 1.05) but failed on time-in-market

A **short-term mean-reversion layer within a momentum regime** (trade reversions only when MaGatedRotation's rotating asset class is in-regime, not flat) would be a genuinely new mechanism combining the confirmed edge of PostShockReversal with the time-in-market guarantees of the rotation framework.
