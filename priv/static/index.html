Overview

What it does

Cripto Trader is an Elixir/OTP application for algorithmic trading on Binance Spot. It cleanly separates exchange integration, strategy logic, and risk controls into independent modules.

◈

Candle Extraction

Fetch historical klines from the Binance REST API or monthly archive files. Supports CSV and JSON output.

▶

Simulation Engine

Deterministic replay of historical candles across multiple symbols with a pluggable strategy function.

◆

Live / Paper Trading

Polling-based trading loop that fetches real-time candles and routes orders through the same pipeline as simulation.

■

Risk Management

Position sizing limits, max drawdown checks, and circuit breakers. Orders are rejected before they hit Binance if they violate rules.

Getting Started

Setup

Prerequisites: Elixir 1.19+ and a compatible Erlang/OTP release.

Clone & install dependencies

$ git clone <repo-url> cripto_trader
$ cd cripto_trader
$ mix deps.get

Create your .env file

The app reads .env in dev/test. It never overwrites variables already set in your shell.

# .env
BINANCE_API_KEY="your-key"
BINANCE_API_SECRET="your-secret"
TRADING_MODE="paper"

Compile & run tests

$ mix compile
$ mix test

Data Extraction

Fetch Candles

Pull historical kline data from Binance REST API or monthly archive files.

REST source

$ mix binance.fetch_candles \
    --symbol BTCUSDT \
    --interval 15m \
    --limit 500

Archive source (bulk historical)

$ mix binance.fetch_candles \
    --source archive \
    --symbols BTCUSDT,ETHUSDT \
    --interval 15m \
    --start-time 2024-01-01T00:00:00Z \
    --end-time 2024-03-31T23:59:59Z
FlagDescription
--symbol / --symbolsOne or more trading pairs (repeatable or comma-separated)
--sourcerest (default) or archive
--formatjson (default) or csv
--intervalCandle interval: 1m, 5m, 15m, 1h, etc.
--start-timeStart bound (ISO8601 or Unix ms)
--end-timeEnd bound (ISO8601 or Unix ms)
--limit1–1000 candles per request (REST only)
ℹ

Archive mode downloads Binance's monthly kline ZIP files and requires both --start-time and --end-time.

Backtesting

Simulation

Replay historical candles deterministically to evaluate strategy performance before risking real capital.

$ mix binance.simulate \
    --source archive \
    --symbols BTCUSDT,ETHUSDT \
    --interval 15m \
    --start-time 2024-01-01T00:00:00Z \
    --end-time 2024-03-31T23:59:59Z \
    --speed 100 \
    --strategy alternating \
    --quantity 0.1 \
    --initial-balance 10000
FlagDescriptionDefault
--speedReplay acceleration factor100
--strategyStrategy module to usealternating
--quantityOrder size per trade0.1
--initial-balanceStarting quote balance10000
--modepaper or livepaper
--include-equity-curveInclude per-event equity pointsoff
--log-strategy-decisionsEnable debug decision logsoff

Live Execution

Trading Robot

A polling loop that fetches live candles and routes strategy orders through the full pipeline. Same strategy function used in simulation.

$ mix binance.trade \
    --symbols BTCUSDT,ETHUSDT \
    --interval 1m \
    --iterations 5 \
    --strategy alternating \
    --quantity 0.1 \
    --mode paper
FlagDescriptionDefault
--iterationsNumber of polling cycles1
--poll-msSleep between iterations (ms)0
--limitCandles fetched per symbol per cycle1
--modepaper or livepaper
⚠

Live mode submits real orders to Binance. Always validate your strategy in paper mode first. The bot defaults to paper to prevent accidental live trades.

Performance

Simulation Benchmark

Deterministic throughput benchmark: 3 symbols × 90 days × 15m candles. Must complete under 5 minutes.

$ mix binance.simulation_benchmark
FlagDescriptionDefault
--symbolsBenchmark symbolsBTCUSDT,ETHUSDT,SOLUSDT
--daysLookback period90
--max-secondsPass/fail threshold300
--speedReplay speed factor100

Outputs JSON with elapsed time, threshold, pass/fail, event count, and simulation summary. Exits with an error if runtime exceeds the threshold.

Self-Improving

Improvement Loop

A file-backed autonomous loop that identifies requirement gaps and optionally invokes an AI agent (Codex) to implement fixes.

Core Commands

# Seed tasks from requirements
$ mix improvement.tasks.seed_requirements

# Manage tasks
$ mix improvement.task.list
$ mix improvement.task.new --title "Investigate X" --type note
$ mix improvement.task.update --id 1 --status in_progress

# Run the loop
$ mix improvement.loop.run --max 5

# Check budget & findings
$ mix improvement.budget.status
$ mix improvement.findings.list --limit 20

# Record architecture decisions
$ mix improvement.decision.new --title "Choose event model"

Autonomous Mode

Run unattended iterations with AI-powered implementation:

$ mix improvement.loop.autorun \
    --iterations 100 \
    --sleep-ms 300 \
    --max-tasks 10 \
    --codex-enabled
ℹ

Budget controls: Execution is capped at 5 hours/week (18,000 seconds). Override with IMPROVEMENT_WEEKLY_BUDGET_SECONDS. The loop pauses automatically when exhausted and resumes after the weekly reset.

Risk Controls

Safety

Trading is inherently risky. This project includes multiple layers of protection, but nothing replaces caution.

✓

Paper by Default

All commands default to paper mode. You must explicitly opt into live.

✓

Max Order Size

Configure MAX_ORDER_QUOTE to cap the quote value of any single order.

✓

Drawdown Breaker

Set MAX_DRAWDOWN_PCT to halt trading when equity drops beyond your threshold.

⚠

Always test in paper mode before using real funds. Review strategy performance in simulation across multiple market conditions. No software can guarantee profits.
